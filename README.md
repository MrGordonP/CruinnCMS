# CruinnCMS

A lightweight, framework-free PHP CMS built for organisations that need full control over their content, site structure, and deployment.

> **v1.0.0-beta.1** — Feature-complete core. Suitable for controlled production deployments. Module system in active development.

---

## What it is

CruinnCMS is a block-based content management system built on custom PHP with no framework dependencies. It provides:

- **Block-based page editor** — every element on a page is individually selectable, editable, and reorderable. Import any existing HTML page and edit it block by block.
- **Multi-instance platform** — one CruinnCMS install can host multiple independent sites. Each instance has its own database, users, and content.
- **Platform dashboard** (`/cms/`) — a top-level admin layer above all instances, with its own credential, instance provisioning, and a built-in DB browser.
- **Template-based pages** — pages can be PHP templates, pure HTML files, or block-composed Cruinn pages.
- **13 built-in block types** — `text`, `heading`, `image`, `gallery`, `html`, `section`, `columns`, `site-logo`, `site-title`, `nav-menu`, `map`, `event-list`, `php-include`
- **Pluggable block registry** — add new block types by dropping in a PHP definition and a JS file, no core changes needed.

## Requirements

- PHP 8.2+
- MySQL 8.0+
- Nginx (config included at `config/nginx.conf`) or Apache with `mod_rewrite`
- Composer

## Quick start

```bash
git clone https://github.com/MrGordonP/CruinnCMS.git
cd CruinnCMS
composer install
```

Point your web server root at `public/`, then visit `/cms/install` to run the install wizard.

See [SETUP.md](SETUP.md) for the full setup guide including database creation, Nginx configuration, and first instance provisioning.

## Architecture

```
public/          Web root (index.php front controller)
src/             PHP engine (Cruinn\ namespace, PSR-4)
templates/       PHP view templates
config/          Configuration (routes.php, config.php, nginx.conf)
schema/          SQL schema (platform.sql, instance_core.sql)
instance/        Per-instance config and state (gitignored at runtime)
tools/           CLI utilities (migrations, email queue, importer)
```

No build step. No JS framework. No ORM. Custom router (~200 lines), PDO + MySQL, vanilla JS, CSS Grid/Flexbox.

## Configuration

Copy `config/config.local.example.php` to `config/config.local.php` and fill in your database credentials. The install wizard at `/cms/install` will write this file for you if you prefer.

Platform credentials are stored in `config/CruinnCMS.php` — generated by the install wizard, never committed to version control.
