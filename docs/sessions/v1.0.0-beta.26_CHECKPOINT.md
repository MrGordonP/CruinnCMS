## Changes

### Admin Settings — Database Table Browser

**Files:**
- `CruinnCMS/src/Admin/Controllers/AcpSystemController.php`
- `CruinnCMS/templates/admin/settings/database-browse.php`
- `public_html/js/admin/database-browse.js`
- `public_html/css/admin-acp.css`

**What changed:**

1. **Column sorting** — clicking a column header sorts ascending/descending, sort state preserved across pagination and filter changes.

2. **Column filters** — per-column LIKE filter inputs rendered below the header row; active filters highlighted; preserved in pagination, sort, and save redirects.

3. **Datasheet edit mode** — "Edit Table" toggle switches the table to an inline editing mode. Each cell shows a hidden `<input>` over a ghost `<span>` (used as sizer so column widths don't jump). Per-row Save / Revert / Delete action buttons appear in edit mode and are hidden in view mode. Save and Delete use `XMLHttpRequest` detection in the controller to return JSON instead of redirecting.

4. **Dirty-row tracking** — rows with unsaved changes receive a `db-row-dirty` CSS class (yellow outline). The Save All button badge updates in real time.

5. **Save All** — sends dirty rows sequentially via `fetch`, shows blue/green/red outline feedback per row, then reports "N rows saved" summary.

6. **Fit-to-content column widths** — `width:1%; min-width:0` on each cell combined with the ghost span sizer keeps columns compact without reflowing on edit.

7. **ACP layout-wide fix** — added `html.admin-layout-wide .acp-wrapper { max-width:none }` so the layout width toggle works on ACP pages.

8. **JS syntax error fix** — previous `cat` heredoc write was corrupted, leaving a duplicate IIFE body in `database-browse.js` causing `Illegal return statement` at line 244. File was cleanly overwritten via `create_file` and verified with `node --check`.

### Membership — Plans Page

**File:** `CruinnCMS/modules/membership/templates/admin/membership/plans/index.php`
**File:** `CruinnCMS/modules/membership/src/Controllers/MembershipAdminController.php`
**File:** `public_html/css/admin-panel-layout.css`

**What changed:**

1. **ID column** — plan ID added as first column in the centre table and shown in the right-panel detail view.

2. **Collapsible groups — left nav** — subscription groups in the left nav are now `<details>`/`<summary>` elements. The group containing the currently selected plan is opened by default; all others are collapsed.

3. **Collapsible groups — centre table** — group rows in the centre table have a `▶`/`▼` toggle button; tier rows are hidden until their parent group is expanded. `togglePlanGroup(id)` JS function handles this client-side.

4. **Group sub-count aggregation** — the controller now sums child tier subscription counts into the group's entry in `$subCountByPlan` so the group badge reflects the total across all its tiers, not just direct subscribers.

### Membership — Members Page

**File:** `CruinnCMS/modules/membership/templates/admin/membership/members/index.php`
**File:** `CruinnCMS/modules/membership/src/Controllers/MembershipAdminController.php`

**What changed:**

1. **All groups now shown** — previously `allPlans(true)` (active only) was used to build the nav tree, so inactive groups never appeared. The controller now fetches `allPlans()` (all plans) separately for tree building while keeping `allPlans(true)` for the subscription form dropdown (unchanged behaviour there).

2. **Collapsible groups with counts** — active subscription groups in the left nav are now collapsible `<details>` matching the plans page style. Each group shows a subscriber count badge. The group containing the current selection is auto-opened.

3. **Standalone plans show counts** — active standalone plans in the nav also display their subscriber count badge.

4. **Inactive groups section** — a collapsed "Inactive Groups" `<details>` block appears below the active groups. Each inactive group is itself a nested collapsible. No counts are shown for inactive groups (no active subscriptions expected). Inactive standalone plans are listed inside the same block.

5. **Sub-count aggregation** — same child-tier-to-group aggregation applied as on the plans page.
