## What changed

### New: `modules/documents/` — standalone document management module

Document management was extracted from the organisation module into its own module.

**Files created:**
- `modules/documents/migrations/001_documents_tables.sql` — `documents` + `document_versions` tables (moved from organisation migration)
- `modules/documents/module.php` — slug=`documents`, provides=[`documents`], ACP section group=Organisation, dashboard widget, 12 routes at `/documents/*`
- `modules/documents/src/Controllers/DocumentController.php` — `index`, `show`, `uploadForm`, `upload`, `newVersion`, `submit`, `approve`, `archive`, `download`, `downloadVersion`, `delete`; uses `renderAdmin` (standard admin layout)

- `modules/documents/templates/admin/documents/index.php`
- `modules/documents/templates/admin/documents/show.php`
- `modules/documents/templates/admin/documents/upload.php`
- `modules/documents/templates/admin/documents/widgets/summary.php`

### Modified: `modules/organisation/`

- `migrations/001_organisation_tables.sql` — removed `documents` + `document_versions` tables (now in documents module)
- `module.php` — removed 12 document routes; description updated to "discussions and inbox only"
- `src/Controllers/OrganisationController.php` — removed all `document*` methods (~400 lines); removed `handleDocumentUpload`, `formatFileSize`, `publicPath` helpers; dashboard fallback now soft-queries documents table via try/catch
- `src/Services/OrganisationDashboardService.php` — document queries wrapped in try/catch (soft cross-module dependency, graceful if documents module not installed)
- `templates/organisation/dashboard.php` — `/organisation/documents/*` links → `/documents/*`
- `templates/admin/organisation/widgets/summary.php` — same URL update

### Modified: `src/App.php`

Added `/documents` prefix middleware entry: `Auth::organisationMiddleware` (same access level as `/organisation`).

---

## Design decisions

- **URL:** `/organisation/documents/*` → `/documents/*` (documents is now a peer module, not a sub-path of organisation)
- **Access control:** Documents routes protected by `organisationMiddleware` (roleLevel >= 50 or role='organisation'), same as before
- **Layout:** DocumentController uses `renderAdmin` (standard admin layout), not the organisation layout — documents accessible from admin panel directly
- **Soft dependency:** Organisation dashboard and OrganisationDashboardService retain a try/catch query on the `documents` table. If the documents module is not installed the organisation workspace degrades gracefully (zeros shown)
- **Module coupling:** The `documents` module has no dependency on `organisation`; organisation has a soft dependency on the documents table only
