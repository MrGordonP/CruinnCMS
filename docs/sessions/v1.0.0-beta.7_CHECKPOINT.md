## Session 2 — 2026-04-19

### Platform fixes (engine-level)

**`Auth::getFlash(string $type): string`** added to `src/Auth.php`
- Pops and returns the first flash message of a given type from `$_SESSION['flash_messages']`
- Previously only `getFlashes(): array` existed; all module templates call `getFlash($type)` individually
- Returns empty string if no message of that type is queued

**`CSRF::verify(): void`** added to `src/CSRF.php`
- Calls `validate()` and aborts with HTTP 403 if token is invalid
- All module controllers call `CSRF::verify()` — previously undefined, causing fatal errors on every POST

**`Template::escape(?string): string`** added to `src/Template.php`
- Alias of existing `e()` method
- All module templates use `$this->escape(...)` — previously undefined, causing fatal errors on every page render
