## 5. Mailbox Module (new)

Full IMAP webmail module: `modules/mailbox/`

**Architecture:**
- IMAP server is permanent archive; DB stores header index only (bodies fetched live)
- Thread graph built from `In-Reply-To` / `References` headers ‚Äî no subject matching
- Per-user read receipts with three-state: unread / partial (some holders read) / all read
- Passwords AES-256-CBC encrypted at rest using instance secret
- Access derived from `organisation_officers.user_id` ‚Äî single login, role-derived mailbox access
- Admin users see all `imap_enabled` mailboxes

**Files created:**
- `module.php` ‚Äî manifest with routes, migrations, acp_sections
- `migrations/001_mailbox_tables.sql` ‚Äî `mailbox_messages`, `mailbox_threads`, `mailbox_reads`, `mailbox_tags`, `mailbox_tag_map`
- `migrations/002_officers_imap_columns.sql` ‚Äî adds IMAP/SMTP credential columns to `organisation_officers`
- `src/Services/MailboxService.php` ‚Äî all IMAP logic: connect, folder list, incremental sync, fetch body, send (PHPMailer), move/delete, threading, read state, tags, IMAP SEARCH, AES-256 encryption helpers
- `src/Controllers/MailboxController.php` ‚Äî public webmail routes `/mail/*`
- `src/Controllers/MailboxAdminController.php` ‚Äî ACP routes `/admin/mailbox/*`
- `templates/mailbox/index.php` ‚Äî mailbox list
- `templates/mailbox/messages.php` ‚Äî three-panel message list with read-state badges
- `templates/mailbox/message.php` ‚Äî single message (sandboxed iframe for HTML body), tags, move/delete, reply
- `templates/mailbox/compose.php` ‚Äî compose / reply / forward form
- `templates/mailbox/search.php` ‚Äî IMAP server-side search results
- `templates/admin/mailbox/index.php` ‚Äî ACP overview with inline sync button

## Data Model Clarification (confirmed this session)

`subscribers` ‚Üî `users` ‚Üî `members` ‚Äî three independent entities. `members.user_id` is an optional bridge. Member rows are never created automatically by user registration or OAuth login. Membership is an explicit act (joining, or admin creation).
