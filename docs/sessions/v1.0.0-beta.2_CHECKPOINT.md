# CruinnCMS v1.0.0-beta.2 — Checkpoint
**Date:** 2026-04-08
**Version:** v1.0.0-beta.2
**Type:** Bug fix / deployment hardening
**Previous:** v1.0.0-beta.1 (`95d8895`)

---

## What changed

### 1. `composer.json` — Remove unused dependencies
Removed three packages that had zero references anywhere in the source:
- `phpoffice/phpword ^1.3`
- `dompdf/dompdf ^3.1`
- `smalot/pdfparser ^2.11`

These were IGAPortal-era remnants. Their presence caused `composer install` to pull ~80 MB of dependencies
and required extensions (`GD`, `fontinfo`) unavailable on many shared hosts, making fresh deployments
fail or time out. `composer install` now generates an autoloader only. The SPL fallback in
`public/index.php` covers deploys without Composer at all.

### 2. `public/.htaccess` — Add `Options -Indexes`
Without this, Apache serves a directory listing of `public/` if `index.php` fails to load — an
information disclosure risk. Added above the existing rewrite block.

### 3. `config/CruinnCMS.example.php` — New file
Documents the structure of the wizard-generated `config/CruinnCMS.php` so operators can reconstruct
it manually after a server migration. Contains correct keys: `initialized`, `username`,
`password_hash`, `db`.

### 4. `config/platform.example.php` — Marked deprecated
This file referenced the old `config/platform.php` structure from the IGAPortal prototype. Updated
to a deprecation stub pointing to `config/CruinnCMS.example.php`.

### 5. `SETUP.md` — New §9: cPanel / Shared Hosting deployment
Full step-by-step guide covering:
- Split-directory layout mapped to a real cPanel account
- Which directories go where relative to `CRUINN_ROOT`
- MySQL database setup via cPanel GUI (no SSH)
- Upload strategy (zip + extract for large trees)
- Permission requirements
- 500-error debug tip (temporary `display_errors`)
- mod_rewrite verification

### 6. `README.md` — Version bump + block count correction
- Version badge updated: `v1.0.0-beta.1` → `v1.0.0-beta.2`
- Block type count corrected: 13 → 22 (anchor, document, element, form, inline, list, list-item, table, php-code were present but unlisted)

### 7. `.github/agents/cruinncms.agent.md` — Full rewrite
The agent file contained the entire IGAPortal/RCEnvironment session history and was wrong for this
public repo. Rewritten to reflect:
- Correct repo: `MrGordonP/CruinnCMS` (public, this is the main repo)
- Correct namespace: `Cruinn\` (no IGA\ references)
- No external Composer deps
- All 22 block types
- Accurate platform dashboard documentation (reads `config/CruinnCMS.php`, not `platform.php`)
- cPanel deployment model
- Version-based history (replaces phase-based history)
- Correct session end checklist for this repo
