## Summary

This session completed the payments and finance boundary refactor so payment ownership is consolidated in the Payments module, Membership consumes a payment service boundary, and Organisation finance ingestion reads from canonical payment records.

---

## Changes

### 1) Membership/Payments schema alignment

**Files:**
- `CruinnCMS/modules/membership/migrations/schema.sql`
- `CruinnCMS/modules/membership/migrations/001_membership_core.sql`
- `CruinnCMS/modules/payments/migrations/schema.sql`
- `CruinnCMS/modules/payments/migrations/001_payments_core.sql`

**What changed:**

- Removed `CREATE TABLE membership_payments` from membership migration schemas.
- Replaced removed table definitions with explicit deprecation comments to prevent schema drift.
- Updated Payments module schema files to define canonical `payments` table instead of legacy `payment_transactions`.

**Why:**

- Runtime payment flows already use `payments`; migration files were reintroducing legacy structures and causing drift.

---

### 2) Payments service boundary extraction

**Files:**
- `CruinnCMS/modules/payments/src/Services/PaymentService.php` (new)
- `CruinnCMS/modules/membership/src/Services/MembershipService.php`
- `CruinnCMS/modules/membership/src/Controllers/MembershipAdminController.php`

**What changed:**

- Added `PaymentService` in the Payments module to own payment-domain operations:
  - manual membership payment recording,
  - linking payments to subscriptions,
  - unlinking payments from subscriptions.
- Updated `MembershipService::recordPayment()` to delegate to `PaymentService`.
- Added membership service wrappers for link/unlink operations.
- Updated membership admin link/unlink controller action to call service boundary methods with error handling.

**Why:**

- Restores module ownership boundaries: Membership should orchestrate subscription workflows, while Payments owns payment mutation logic.

---

### 3) Organisation finance ingest source correction

**File:**
- `CruinnCMS/modules/organisation/src/Services/FinanceService.php`

**What changed:**

- Membership payment ingest now reads completed rows from canonical `payments` joined to `membership_subscriptions`/`members`.
- Ledger reference now uses `transaction_id`.

**Why:**

- Prevents ingestion from querying deprecated `membership_payments` and aligns finance reporting with the active payment runtime model.

---

### 4) Module dashboard/summary metric cleanup

**File:**
- `CruinnCMS/src/Modules/ModuleRegistry.php`

**What changed:**

- Updated Membership and Payments summary queries to use `payments`.
- Removed `payment_transactions` references.
- Updated Payments exception stat from legacy “Unverified” to canonical “Unlinked” (`subscription_id IS NULL`).

**Why:**

- Dashboard metrics now reflect canonical data model and avoid stale table assumptions.
