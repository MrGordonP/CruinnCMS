## Route and Flow Changes

### New top-level entry

- Added hub route:
  - `GET /admin/membership -> MembershipAdminController::hub()`

### Members workspace moved

- Previous:
  - `GET /admin/membership -> indexMembers()`
- Now:
  - `GET /admin/membership/members -> indexMembers()`

### Redirect harmonization

Membership controller actions that previously redirected to `/admin/membership` for member context now redirect to `/admin/membership/members` (often with `?member={id}`), including:

- member create/update/show/edit shortcuts
- subscription creation/status updates
- payment recording fallbacks
- user link/unlink flows
- import breadcrumbs and navigation context updates

This keeps the hub as a stable landing page and the three-panel members workspace as the operational detail area.

---

## Membership Hub Details

### Controller (`hub()`)

`MembershipAdminController::hub()` gathers high-level counts for:

- member statuses
- plans
- subscriptions
- payments with subscription linkage
- forms
- form submissions
- pending submissions
- latest form metadata (for quick latest response link)

All count/read operations are guarded with `try/catch` and safe-zero fallback semantics.

### Template (`admin/membership/hub`)

Hub provides four card areas:

- Membership Lists
- Subscriptions and Plans
- Forms and Responses
- Import

Final Forms/Responses implementation is:

- direct Forms navigation (`/admin/forms`)
- optional latest responses link to latest form submissions
- explanatory copy that forms/responses are linked via Subject associations

This satisfies: use Subjects integration semantics without routing users into Subjects workspace.

## Follow-up Addendum (Panel Shell Standardization)

### Why this follow-up was required

The broad beta.19 panel migration introduced many `panel-layout` pages using `pl-sidebar` / `pl-detail` wrappers instead of the standardized `pl-panel` family shell used by the editor implementation. This reduced consistency of collapse behavior and diverged from the intended shared panel contract.

### What was corrected

- Shared collapse bootstrap now binds from standard DOM shape (`.panel-layout` with direct panel children) in:
  - `public_html/js/admin/panel-collapse.js`
- Converted migrated no-detail pages from `pl-sidebar` shell to `pl-panel pl-panel-left` + `pl-panel-header` + `pl-panel-body`.
- Preserved existing inner content/actions; only outer panel-shell classes/containers were adjusted.
