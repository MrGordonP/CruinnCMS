# v1.0.0-beta.22 Session Checkpoint

## Purpose

Stabilize the membership plans workspace after grouped tiers rollout, resolve platform login fault handling regressions, and complete the semantic split between shared subscriptions and structural plan groups.

---

## Scope

### Included

- Membership plans/admin stabilization:
  - surfaced plan save failures in the form
  - improved plans listing/group rendering consistency
  - fixed tier creation flow regressions tied to group classification
- Membership domain model correction:
  - preserved `is_group` for shared-subscription semantics (multiple members)
  - introduced `is_plan_group` for structural hierarchy semantics
  - updated controller/service/template logic to use the correct field per concern
- Membership schema updates:
  - promotion columns migration (`035_membership_plan_promotions.sql`)
  - structural group column migration (`036_membership_structural_groups.sql`)
  - legacy-safe migration guards and index creation patterns
- Platform auth hardening:
  - defensive credential load handling in platform auth path to avoid hard-fatal startup conditions

### Explicitly Not Included

- No new dependencies
- No instance-specific code paths
- No router architecture redesign
- No redesign of member subscription lifecycle

## Migration Notes

### 035: membership plan promotions

Adds promo metadata columns on `membership_plans` for time-bound discounts:

- `promo_type`
- `promo_value`
- `promo_starts_at`
- `promo_ends_at`

Migration follows guarded/idempotent style compatible with repeated runs.

### 036: membership structural groups

Adds structural hierarchy flag to `membership_plans`:

- `is_plan_group` (`TINYINT(1) NOT NULL DEFAULT 0`)
- index: `idx_membership_plans_structural_group`

Backfills existing structural groups using legacy heuristics based on prior grouped-plan behavior.

## Behavioral Outcome

- Group vs tier logic now tracks explicit structural intent using `is_plan_group`.
- Shared subscription behavior remains driven by `is_group` and `max_members` rules.
- `max_members=0` is accepted as unlimited for shared subscriptions.
- Plan/tier forms now better surface validation and save failures to avoid silent-looking submits.
- Plans list/actions align better with grouped hierarchy state.
- Platform auth initialization is more resilient against config load failure conditions.
