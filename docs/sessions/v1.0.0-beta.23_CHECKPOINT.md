## What changed

### Membership — Account Merge
- Added "Merge accounts (select exactly 2)…" option to members bulk action bar
- New routes: `GET /admin/membership/members/merge`, `POST /admin/membership/members/merge`
- `MembershipService::mergeInto()` — repoints subscriptions, payments, address (if primary has none), user_id (if primary has none), then deletes secondary
- New merge confirmation template (`members/merge.php`) showing both members side-by-side, radio to choose primary
- `bulkMembers()` gains `merge` case validating exactly 2 selected IDs
- Fixed `joined_at` NULL guard in merge template (hotfix)

### Admin sidebar
- `AdminSidebarMenuService` rewritten: each module is its own flat or flyout entry under a "Modules" section heading
- Single-child modules render as flat links (no pointless flyout)
- "People" renamed to "Accounts" (Users, Roles, Groups only — Membership removed)
- Accounts moved above the Modules section with the other core items
- New `admin-sidebar-section` CSS class for the heading label
- Layout template updated to render `type: heading` items

### Roles fix
- Removed duplicate `<?php endif; ?>` in `templates/admin/roles/edit.php` (ParseError)

### Template cleanup
- Deleted 13 stale platform-layer copies that were shadowing module templates (events ×3, forms ×2, gdpr ×3, members/directory, members/profile, articles/sidebar)
- `subjects/show.php` moved from `templates/public/subjects/` → `templates/subjects/` and render call updated in SubjectController

### Notifications module (new)
- New module: `modules/notifications/`
- Schema: `notifications` table (user inbox), `notification_preferences` table (per-category delivery prefs)
- `NotificationService` — inbox CRUD, unread count, preferences, mailing list subscribe/unsubscribe (by user + by token)
- `NotificationsController` — 9 public routes
- Templates moved from `templates/public/notifications/` into the module
- Routes: `/notifications`, `/notifications/preferences`, `/notifications/{id}/read`, `/notifications/read-all`, `/mailing-lists`, `/mailing-lists/{id}/subscribe`, `/mailing-lists/{id}/unsubscribe`, `/unsubscribe` (token)
