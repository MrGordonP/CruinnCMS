# Session Checkpoint — v1.0.0-beta.28

**Date:** 2026-06-12
**Branch:** `main`

---

## Summary

Platform shell refactor: inverted to a fixed app frame with unconditional geometry, eliminated context-dependent :has() overrides, unified cache-busting via asset() helper, and normalized footer/width-toggle behaviour.

---

## Changes

### 1) Platform shell inversion — fixed app frame

**Files:**
- `public_html/css/platform.css`
- `CruinnCMS/templates/platform/layout.php`

**What changed:**

- `html` and `.platform-body` → `height:100%; overflow:hidden` — fixed viewport frame, always
- `.platform-content` deleted; `.platform-main` is now the single scroll region (`flex:1; min-height:0; overflow-y:auto`)
- Both `:has(#editor-wrap)` and `:has(#source-wrap)` CSS override stacks deleted (~110 lines)
- `#source-wrap` reduced to width vars + `height:100%` — `.panel-layout` does the rest
- Padding moved off the shell onto `.platform-page` wrapper (flat document pages already use it)
- `.platform-editor-page` body class and associated styles removed — no context checks
- `.platform-bar` no longer `position:sticky` (meaningless in a fixed frame)

**Why:**

The source/editor pages proved the fixed app-frame geometry works. Every other page was falling back to legacy document-flow shell where the sidebar scrolled away with page content and the footer sat below the fold. This makes the working geometry unconditional: sidebar nav scrolls internally with pinned hero/footer, topbar fixed, `.platform-main` is the scroll region for flat pages, panel layouts fill it exactly and scroll internally.

Zero context awareness — the shell never inspects its contents; components bring their own geometry.

### 2) Footer relocation and collapse

**Files:**
- `CruinnCMS/templates/platform/layout.php`
- `public_html/css/platform.css`
- `public_html/js/platform/shell.js`
- `public_html/js/platform/boot.js`

**What changed:**

- Footer moved inside `.platform-right` after `.platform-main` — sidebar menu never obscured
- Footer toggle button added (▾/▴ chevron, `pl-panel-toggle` idiom), persisted as `platform-footer-collapsed` in localStorage
- Collapsed state: footer shrinks to 16px strip with toggle only
- boot.js pre-applies collapsed state to avoid flash

**Why:**

Footer was outside the shell, pushing the sidebar menu below viewport when pages were tall. Now it's a fixed row inside the right column, always visible, hideable via toggle for max content space.

### 3) Width toggle — device preview modes

**Files:**
- `public_html/css/platform.css`
- `public_html/js/platform/shell.js`
- `public_html/js/platform/boot.js`

**What changed:**

- Width button cycles desktop (unconstrained) → tablet (768px) → mobile (390px)
- Modes constrain `.platform-main` width via `max-width` + `margin-inline:auto`
- Persisted as `platform-width-mode` (desktop/tablet/mobile) in localStorage
- `platform-layout-wide` class retired

**Why:**

Previous toggle was a frozen on/off between two hard-coded constraints. New behaviour gives actual device-standard preview widths for responsive testing, matching the mental model of what a "width toggle" should do.

### 4) asset() helper — unified cache-busting

**Files:**
- `CruinnCMS/src/Template.php` (new helper)
- `CruinnCMS/templates/platform/layout.php`
- `CruinnCMS/templates/platform/login.php`
- `CruinnCMS/templates/platform/install.php`
- `CruinnCMS/templates/admin/layout.php`

**What changed:**

- Added `asset(string $path): string` global helper in Template.php
- Returns `url($path) . '?v=' . filemtime(CRUINN_PUBLIC . $path)`
- Swept all layout heads onto it: platform (4 assets), admin (base CSS, queued CSS, boot.js, 12 admin modules, queued JS), login, install
- Deleted inline `file_exists() ? filemtime() : 0` repetition and the `$adminJsBase`/`$mtime` scaffolding
- Fixed login.php and install.php which had bare unversioned CSS links (worst offenders)

**Why:**

Cache-busting was split between frozen manual tokens (`?v=beta6`), inline `filemtime()` expressions, and completely unversioned links. Single helper provides one source of truth; future switch to content hashing would be a one-line change. Derivative truth, zero discipline required.

### 5) Token/class cleanup

**Files:**

- `CruinnCMS/templates/platform/source-editor.php`

**What changed:**

- Normalized migrations.php to canonical `.platform-page` pattern
- Replaced dead `plat-content-*` / `plat-btn` classes with canonical equivalents

**Why:**

Drift cleanup — undefined tokens and dead classes were scattered across templates from partial refactors. Polish pass = no tolerance for these.

---

### 6) Login/install page scroll fix

**Files:**
- `public_html/css/platform.css`

**What changed:**

- `.platform-login-body` → `display:flex; overflow-y:auto` (body becomes scroll region)
- `.platform-login-wrap` → `margin:auto` (card centres in flex, scrolls when tall)

**Why:**

Standalone pages (no `.platform-wrap` shell) were `min-height:100vh` with `align-items:center`, which clips tall forms on short viewports. Auto margins let the card centre naturally while still scrolling when needed.

## Net effect

- **Dashboard, settings, migrations, database, provision:** all gain source-page behaviour automatically — fixed sidebar, fixed topbar, visible hideable footer, content-only scrolling, no footer below fold
- **Source/editor pages:** render identically to before, minus their special-casing
- **Login/install:** tall forms now scroll instead of clipping
- **All pages:** cache-busted assets via one helper, width preview modes, footer collapse
- **Codebase:** ~120 lines of context-check CSS deleted, zero layout flags, zero `:has()` queries, components own their geometry
