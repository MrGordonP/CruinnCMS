## Changes Since beta.4

All changes in this release are bug fixes and one small feature discovered missing during first live deployment on gordonpoole.ie.

### Bug Fixes

**Deployment / Provisioning**
- `fix(platform)`: Remove orphaned old provision template body causing double-render (`6f403b2`)
- `fix(platform)`: Write `hostname` key to `instance/{slug}/config.php` on provisioning — without it, the instance was never matched by hostname routing (`f387129`)
- `fix(platform)`: Normalise `INFORMATION_SCHEMA` column name casing in DB browser (`TABLE_NAME` vs `table_name`) — MySQL on some hosts returns uppercase (`f1c3fe8`)

**Admin Templates — Missing PHP Opening Tags**
- `fix(admin)`: Missing `<?php` in `templates/admin/settings/security.php` caused PHP code to render as text (`d39ffe0`)
- `fix(admin)`: Missing `<?php` in `templates/admin/users/edit.php` and `templates/admin/groups/edit.php` (`f8237ca`)
- `fix(admin)`: Missing `<?php` in `templates/admin/roles/dashboard-config.php` and `templates/admin/roles/navigation-config.php` (`16c84e9`)

**Admin Templates — Other**
- `fix(admin)`: `admin-members.css` referenced in users/groups templates but file never existed; changed to `admin-base.css` (`138a8cc`)
- `fix(admin)`: Remove unimplemented `email_verified_at` row from `templates/admin/users/show.php` — column did not exist in schema (`3ce73d0`)

**Schema**
- `feat(users)`: Add `email_verified_at DATETIME NULL` column to `users` table in `schema/instance_core.sql`; reinstate Email Verified row in users/show template (`0d7f044`)

### Features

- `feat(platform)`: Reset instance user password from platform dashboard — new route, controller method, and UI form (`60311d1`)
- `feat(users)`: Add `/users/profile` page — GET shows logged-in user's profile; POST updates display name and optionally password (`b6da891`, `722627b`)
