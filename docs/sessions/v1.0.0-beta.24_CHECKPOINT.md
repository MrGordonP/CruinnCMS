## What Changed

### `public_html/js/admin/membership.js` (new file)

Replaced all inline `<script>` blocks across membership admin templates with a single external JS file loaded via `Template::requireJs('membership.js')`.

- Bulk selection bar (`updateBulkBar`) driven by event delegation on `.member-cb`
- Subscription sub-row toggle (`toggleSubRow`) via `.sub-expand-btn[data-mid]`
- Row navigation via `data-row-url` (single click) and `data-row-profile-url` (double click)
- Confirmation dialogs via `data-confirm` attribute
- Deselect-all via `data-action="deselect-all"`
- Scroll selected subscription row into view on page load (`tr.selected` inside `.pl-main-scroll`)

### `members/show.php` — rebuilt as member hub

`/admin/membership/members/{id}` and `/admin/membership/members/{id}/edit` are now the same page.

**Left panel — editable form (POST to `/{id}`):**
- Forenames, surnames, email, membership #, status (select), organisation, joined/lapsed dates
- Full address block: line 1/2, city, county, postcode, country, phone
- Save + Back to Members buttons
- Validation errors displayed inline

**Centre panel:**
- Subscriptions table (period, plan, amount, method, tx ID, verification badge, View link)
- Add Subscription form

**Right panel:**
- Admin Notes — textarea posting to `/{id}` with hidden fields carrying current member state
- Linked User Account — shows display name + email with "View user account" link and Unlink button; or search-by-email/display-name form to link one
- Record Payment form (subscription selector, amount, currency, tx ID, gateway, paid_at)
- Payment History list

### `MembershipAdminController.php`

- `showMember()` — now fetches `$memberAdmin` from `member_admin` table and passes `$errors = []`
- `editMember()` — simplified to redirect to `/{id}`
- `updateMember()` — saves member + address (`saveAddress`) + admin notes (`upsertMemberAdmin`) in one POST; on validation error re-renders the hub
- `memberPayload()` extended with phone, status, joined_at, lapsed_at
- New private helpers: `addressPayload()`, `adminNotesPayload()`
- `linkUser()` / `unlinkUser()` redirects corrected to `/admin/membership/members/{id}`
- `createMember()` redirect corrected to `/admin/membership/members/{id}`

---

### `MembershipService.php`

- `updateMember()` now saves status, joined_at, lapsed_at
- New `memberAdminForMember(int $memberId)` — fetch `member_admin` row
- New `upsertMemberAdmin(int $memberId, array $data)` — insert or update admin notes
- New `saveAddress(int $memberId, array $data)` — insert or update `member_addresses` including phone

**Note:** The method was originally named `upsertAddress` which collided with a pre-existing private `upsertAddress` in the CSV import logic. Renamed to `saveAddress` (`49fabf3`). When deploying `MembershipService.php`, pull the containing **folder** (`modules/membership/src/Services/`), not the individual file path — the server-side deployment tool does not invalidate its file cache on single-file pulls.
