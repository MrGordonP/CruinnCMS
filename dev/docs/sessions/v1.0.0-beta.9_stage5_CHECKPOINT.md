## What Changed

Completed Stage 5 of the role capability refactor: created a provider-based widget system for dashboard canvases and implemented the mailbox notifications widget as the first example.

### Core Enhancement — Provider-Based Widgets

**Problem:** The existing module-widget block only supported simple widgets (pre-rendered HTML) from the `widgets` callable. Dashboard canvases need widgets that can query user-specific, position-specific data using the injected `_userContext`.

**Solution:** Added a `widget_providers` registration pattern to modules, allowing widgets to define a data provider (static method) + template path. The module-widget block renderer now detects `_userContext` in config and calls the provider instead of returning pre-rendered HTML.

### Modified: `src/Modules/ModuleRegistry.php`

**Added:**
- `widget_providers` key to module definition defaults
- `renderProviderWidget(string $widgetKey, array $settings, array $userContext): string` method

**How it works:**
1. Parses widget key (`module:slug`)
2. Looks up widget definition in module's `widget_providers` array
3. Calls provider (format: `"Class::method"`) with `($settings, $userContext)`
4. Renders template from module's `template_path` with extracted data variables
5. Falls back to `renderWidgetByKey()` if no provider found

### Modified: `src/BlockTypes/module-widget/definition.php`

**Enhanced renderer:**
```php
$userContext = $config['_userContext'] ?? null;

if ($userContext && is_array($userContext)) {
    // Provider-based widget — extract settings and call provider
    $settings = $config;
    unset($settings['widget_key'], $settings['_userContext']);
    $html = ModuleRegistry::renderProviderWidget($widgetKey, $settings, $userContext);
} else {
    // Simple widget (sidebar) — pre-rendered HTML
    $html = ModuleRegistry::renderWidgetByKey($widgetKey);
}
```

Backward compatible: simple widgets (no userContext) work as before.

### New File: `modules/mailbox/src/Widgets/NotificationsWidget.php`

Data provider class for mailbox notifications widget.

**Method:** `getData(array $settings, array $userContext): array`

**Queries:**
1. User ID + role level from `$userContext`
2. Accessible mailboxes via `MailboxService::getAccessibleMailboxes()`
3. For each mailbox:
   - Count total messages in INBOX (`mailbox_messages`)
   - Count read messages for user (`mailbox_reads`)
   - Calculate `unread_count = total - read`
4. Return `['mailboxes' => [...], 'total_unread' => int]`

**Access control:** Uses `MailboxService` which enforces position-based and admin-level access to mailboxes.

### New File: `modules/mailbox/templates/widgets/notifications.php`

Widget template partial that renders:
- Widget title with total unread badge
- List of accessible mailboxes with individual unread counts
- Links to `/mail/{mailbox_id}`
- "View All Mailboxes" footer link
- Scoped CSS (no external dependencies)

**Design:** Clean, badge-based UI with hover states and responsive layout.

### Modified: `modules/mailbox/module.php`

**Added:**
```php
'widget_providers' => [
    [
        'slug'     => 'notifications',
        'label'    => 'Mailbox Notifications',

        'template' => 'widgets/notifications',
    ],

```

Widget key for editor selection: `mailbox:notifications`

## How to Use

### For Admins (Creating a Dashboard)

1. Navigate to `/admin/site-builder/dashboards`
2. Create a new widget dashboard (e.g., "Council Dashboard")
3. In the block editor, add a `Module Widget` block
4. In Content properties, select widget: `Mailbox — Mailbox Notifications`
5. Publish the dashboard
6. Assign to a role at `/admin/roles/{id}/dashboard-canvas`

### For Developers (Creating New Provider Widgets)

```php
// 1. Create data provider class
class MyWidget
{
    public static function getData(array $settings, array $userContext): array
    {
        $userId = $userContext['user_id'] ?? 0;
        // Query data specific to $userId, position_ids, role_level
        return ['key' => 'value'];
    }
}

// 2. Create template partial (templates/widgets/my-widget.php)
<div class="my-widget">
    <p><?= e($key) ?></p>
</div>

// 3. Register in module.php
'widget_providers' => [
    [
        'slug'     => 'my-widget',
        'label'    => 'My Widget',
        'provider' => 'Cruinn\\Module\\MyModule\\Widgets\\MyWidget::getData',
        'template' => 'widgets/my-widget',
    ],
]
```
