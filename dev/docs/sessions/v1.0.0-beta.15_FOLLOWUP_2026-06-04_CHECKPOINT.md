## Key Behavioral State at Handoff
- Source editor includes lazy host-parent node and loads children via /cms/source/list.
- Host-parent files can be opened/read (including extensionless log files).
- Host-parent save/pull/pull-dir are blocked by read-only guards.
- Linked logs in settings accept both:
  - absolute paths (/home/...)
  - host-parent paths (host-parent/...)
- Linked log View resolves host-parent entries through the source path resolver.

## Current Settings Log UX
Within the linked logs dialog:
- Pick opens an in-settings picker (host-parent start), navigates directories, and writes selected file path into the row (host-parent/...).
- Browse:
  - opens mapped file directly in source editor if path is resolvable to host-parent
  - if row is blank/unmapped, opens the picker flow rather than bouncing to source-editor file errors
- Remove removes the row.
