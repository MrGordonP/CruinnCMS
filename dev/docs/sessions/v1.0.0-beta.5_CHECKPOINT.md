# CruinnCMS v1.0.0-beta.5 — Checkpoint
**Date:** 2026-04-13
**Version:** v1.0.0-beta.5
**Type:** Editor UX improvements — Properties panel collapse, code view clean HTML, CSS class persistence
**Previous:** v1.0.0-beta.4 (`7cfdc83`)

---

## What changed

### 1. Properties panel accordions start collapsed (except Identity)

All accordion groups in the right-hand Properties panel now load collapsed by default. Only the **Identity** group (Block ID, Type, CSS Class, Collapsed checkbox) starts expanded.

**File:** `templates/admin/editor.php`
- Added `collapsed` class to: PHP Code, Typography, Spacing, Image, Site Title, Background, Border, Effects, Size & Position, Layout, Content groups
- Identity group left without `collapsed` class

**Rationale:** Reduces visual overwhelm when selecting a block. Users expand only the panels they need.

### 2. Code view shows clean publishable HTML

Previously, toggling Code View on a block-based page showed the raw editor DOM with `data-block`, `data-block-type`, `data-block-config` attributes — not useful for reviewing output.

Now, Code View serializes blocks to their **publishable HTML output**:
- Block type → correct HTML tag (`section`, `h1`–`h6`, `figure`, `nav`, `ul`, `li`, etc.)
- CSS classes preserved (editor-only classes stripped)
- Inline styles preserved
- Nested blocks indented properly
- PHP code blocks output their raw `<?php ... ?>` source
- Image blocks output `<figure><img></figure>` structure

**File:** `public/js/editor.js`
- Added `BLOCK_TAGS` mapping (mirrors PHP `BlockRegistry` tag definitions)
- Added `blockToHtml(block, indent)` — recursive block-to-HTML serializer
- Added `blocksToHtml()` — serializes entire canvas
- Changed `enterCodeView()` to call `blocksToHtml()` instead of `formatHtml(canvas.innerHTML)`

**Example output:**
```html
<section class="hero" style="padding: 2rem;">
    <h1>Welcome</h1>
    <div>Introductory paragraph here.</div>
</section>

<section class="content">
    <div>More content...</div>
</section>
```

**Note:** Editing code view and toggling back replaces the block structure with raw HTML. This is intentional — code view shows draft output, not block source. Use the Site Builder template editor for source file editing.

### 3. CSS class persistence (from previous session, documenting here)

CSS classes are now properly saved and rendered:
- **Save path:** Editor strips editor-only classes (`active`, `dragging`, `drag-over`, `drag-over-inside`), stores remainder in `css_props._class`
- **Render path:** `CruinnRenderService` and `EditorRenderService` read `css_props._class` and output as `class="..."` attribute

**Files:**
- `public/js/editor.js` — `serialiseCanvas()` extracts classes into `css_props._class`
- `src/Services/CruinnRenderService.php` — outputs class attribute on published pages
- `src/Services/EditorRenderService.php` — outputs class attribute on editor canvas reload

### 4. Collapsed checkbox in Identity panel (from previous session)

Added a **Collapsed** checkbox to the Identity panel. When checked, adds `collapsed` class to the block. This lets site CSS define collapsible sections (e.g., FAQ accordions) — the editor just manages the class, site CSS defines the behaviour.

**Files:**
- `templates/admin/editor.php` — checkbox with hint text
- `public/js/editor.js` — checkbox change handler, class toggle on block element

## Known issues / future work

1. **Code view round-trip** — Editing HTML in code view then toggling back replaces block structure. May want a "read-only preview" mode separate from editable code view.
2. **Heading level in code view** — `blockToHtml()` checks `config.level` for headings but `config` population from block may vary. Needs verification.
3. **Module migrations** — Active reorganization (renumbering to `001_*`). Do not run migrations until stabilized.
4. **Fresh install testing** — Next beta should include a full from-scratch install test.
