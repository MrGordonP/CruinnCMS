# Session Checkpoint — v1.0.0-beta.9 (mobile/tablet nav accordion)

**Date:** 2026-04-28  
**Branch:** `main`

---

## What Changed

### Platform Sidebar: Instance Menu Grouping

**Updated:** `CruinnCMS/templates/platform/layout.php`
- Wrapped instance editor links under an explicit sidebar nav group:
  - `#platform-instance-menu-group`
  - `#platform-instance-menu-toggle`
  - `#platform-instance-menu-body`
- Preserved existing instance link targets and active-state behavior.

**Updated:** `public_html/js/platform/shell.js`
- Added responsive state handling for the new instance group.
- Desktop (`>=1024px`): instance group forced open.
- Mobile/tablet: instance group can be collapsed/expanded.
- Persists mobile/tablet preference in `localStorage` key:
  - `platform-instance-menu-open-mobile`
- Re-syncs group state when crossing desktop/mobile breakpoint.

### Instance Frontend Nav: Tablet/Mobile Vertical + Accordion

**Updated:** `public_html/js/main.js`
- Added submenu enhancement for header nav lists:
  - `.main-nav .nav-list`
  - `.utility-nav`
  - `.site-header-custom .block-nav-menu > ul`
- For items with child `.nav-dropdown`, injects `.nav-submenu-toggle` button.
- Implements interaction model on tablet/mobile (`<=1023px`):
  - First tap on parent link opens submenu.
  - Second tap navigates if href is real.
  - If href is `#`/empty/javascript, second tap closes submenu.
- Accordion behavior: opening one sibling closes others at the same level.
- Outside click closes open submenu trees.
- Desktop resize cleanup closes all open submenu states.

**Updated:** `public_html/css/style.css`
- Added shared tablet/mobile nav styles (`@media (max-width: 1023px)`) to:
  - stack main nav, utility nav, and custom header nav vertically
  - place submenu toggles on the right (`.nav-submenu-toggle`)
  - disable hover-open dropdown behavior on tablet/mobile
  - show nested lists only when `.submenu-open` is present
- Preserved desktop hover dropdown behavior.
- Kept existing 768px visual adjustments (logo/banner text sizing), while moving nav mechanics to 1023px to cover tablets.

---

## Why

The active instance header layout placed main and utility menus in a header block that did not adapt cleanly on tablets/mobile. This checkpoint introduces a consistent, touch-friendly, engine-level behavior so nested navigation remains usable and predictable across smaller breakpoints.
