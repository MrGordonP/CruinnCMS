**Date:** 2026-05-05
**HEAD:** 8cb9700

---

## What Changed This Session

### 1. ThemeController::parseVariables — empty `:root` match fix
- File-header comment contained `:root { }` text which the regex matched before the real `:root` block
- Fix: require the matched block to contain `--` (a CSS custom property), so empty comment mentions are skipped
- Commit: `d279437`

### 2. Media template — duplicate inline JS and stale old implementation removed
- `templates/admin/media/index.php` contained the full JS inline (already extracted to `media.js`) plus the entire old flat media browser implementation below it
- Both removed; template is now 113 lines of HTML only
- Commit: `2d69468`

### 3. Theme editor accordion toggles not working
- Root cause: `editor.js` IIFE bails at `if (!canvas)` because the theme page renders `#theme-preview-root` instead of `#editor-canvas`
- Fix: standalone delegated accordion handler added in the `isThemePage` script block at the bottom of `editor.php`
- Also changed `bindAccordions()` in `editor.js` to use event delegation (belt-and-braces)
- Commits: `29becea`, `c2518ec`

### 4. Theme preview showing Times New Roman
- `#theme-preview-vars` was using `e()` (htmlspecialchars) on CSS values — single quotes in font names became `&#039;Arial&#039;`, which is invalid CSS and fell back to browser default serif
- Fix: removed `e()` from CSS value output in `#theme-preview-vars` (values come from trusted theme file)
- Additionally: preview now loads the active theme CSS via `<link>` tag so all vars are set from the file; `rebuildVars()` seeded on page load for live-edit override
- Commits: `cf28c59`

### 5. Themes nav group in editor left panel
- Controller scans `CRUINN_PUBLIC/css/themes/*.css` and passes `$themeFiles` (sorted slug list) to template
- Left panel gains a "🎨 Themes" nav group (collapsed by default) listing each theme
- Clicking a theme links to `/_typography editor?theme=slug`, reloading the page with that theme's vars
- `?theme=` param validated with regex; falls back to active theme if missing/invalid
- `$editingTheme` passed from controller so template never calls `activeTheme()` independently
- Commits: `62d3743`, `8cb9700`

## Known Pending Issues / TODOs

- **"Set as Active" button** — no way to promote an edited theme to the active site theme yet. Needs a POST to `/admin/theme/activate` writing `site.active_theme` to settings, and a route + `ThemeController::activate()` method.
- **`?theme=` param carried through Save** — when editing a non-default theme, the save form POSTs to `/admin/theme` which redirects to `/admin/theme` (GET → redirect to `_typography` editor) without the `?theme=` param, losing the theme context. The save redirect should preserve the editing theme.
