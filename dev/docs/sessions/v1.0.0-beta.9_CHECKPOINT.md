- `.no-detail` variant for 2-column pages
- Classes: `.pl-sidebar`, `.pl-main`, `.pl-detail` and all sub-elements (toolbar, scroll, table, nav-items, etc.)

**Redesigned `/admin/pages`** (`templates/admin/pages/index.php`)
- Left: filter nav by status and page mode with counts
- Middle: table with JS search/filter; no inline action columns
- Right: detail panel on row click with all actions

**Redesigned `/admin/media`** (`templates/admin/media/index.php`)
- Left: folder tree (AJAX-loaded sub-folders), inline new folder form
- Middle: media grid loaded via AJAX; search; upload button

**Redesigned `/organisation` dashboard** (`modules/organisation/templates/organisation/dashboard.php`)
- Left: nav sidebar (Dashboard, Documents, Discussions, Inbox, Admin Panel)
- Middle: stats grid + Recent Documents + Active Discussions tables
- Right: "At a Glance" meta counts + quick actions

### Mailing Lists UI (new)

**New controller:** `modules/mailout/src/Controllers/MailingListController.php`
- `index` — 3-panel list manager
- `create` — POST new list (name, slug, description, mode, is_public)
- `update` — POST list settings
- `delete` — POST delete list + cascade subscriptions
- `subscribers` — GET JSON subscriber list with search/status filter
- `removeSubscriber` — POST remove a single subscriber (AJAX)

**New template:** `modules/mailout/templates/admin/lists/index.php`
- Left: mailing list nav with subscriber counts; inline new-list form
- Middle: subscriber table loaded via AJAX; search + status filter; inline remove
- Right: settings form (name, description, mode, public/active toggles, delete)

**Updated:** `modules/mailout/module.php`
- Added `MailingListController` use + 6 new routes (registered before `{id}` wildcard)
- Added "Mailing Lists" entry to `acp_sections` and `dashboard_sections`
