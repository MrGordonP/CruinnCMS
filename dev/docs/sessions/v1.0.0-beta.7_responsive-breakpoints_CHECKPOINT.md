## What changed and why

### Problem
Instance headers (and all block-layout zones) used inline pixel widths set by the editor.
On narrow screens these widths overflowed â€” there was no mechanism to define per-breakpoint
styles, and the canvas gave no feedback about how a design would look at smaller sizes.

### Design decisions made
- **Two extra breakpoints** (not one): tablet (600â€“1023 px) and mobile (â‰¤ 599 px).
  Desktop (â‰Ą 1024 px) remains the base â€” no media query emitted.
- **Breakpoints are content-width oriented**, not industry-framework oriented:
  - Mobile â‰¤ 599 px â†’ canvas simulation at 360 px (worst-case phone)
  - Tablet 600â€“1023 px â†’ canvas simulation at 600 px (worst-case tablet)
- **Inheritance: desktop â†’ tablet â†’ mobile cascade** (mobile inherits unset tablet props,
  which inherits desktop). In practice this means only overrides need to be stored.
- **Storage: separate columns** (`css_props_tablet`, `css_props_mobile` JSON NULL) rather
  than namespaced keys inside existing `css_props`. Clean separation, easy querying.
- **All CSS properties are breakpoint-aware** â€” the same properties panel is used at all
  three viewports; it reads/writes into the active breakpoint's dataset.
- **Canvas simulation only constrains max-width** â€” no device frame chrome.

## Files changed

### New
- `CruinnCMS/migrations/core/006_responsive_breakpoints.sql`  
  Adds `css_props_tablet JSON NULL` and `css_props_mobile JSON NULL` to both `pages`
  and `pages_draft`.

### Modified

**`CruinnCMS/src/Services/CruinnRenderService.php`**  
- `buildCss()` now fetches `css_props_tablet` and `css_props_mobile` alongside `css_props`.
- Emits desktop rules as before, then appends `@media (max-width: 1023px)` and
  `@media (max-width: 599px)` blocks for any non-empty breakpoint overrides.
- Added private `buildRules(array $props): string` helper â€” shared rule sanitisation,
  skips internal keys (`_` prefix).

**`CruinnCMS/src/Services/EditorRenderService.php`**  
- `buildCanvasCss()` mirrors the same three-tier CSS output as CruinnRenderService.
- Added matching `buildRules()` private helper.
- `renderTree()` now emits `data-css-props-tablet` and `data-css-props-mobile` HTML
  attributes on canvas block elements (only when non-empty, same pattern as `data-css-props`).

**`CruinnCMS/src/Controllers/CruinnController.php`**  
- `recordAction()` input parsing: reads `css_props_tablet` and `css_props_mobile` from
  incoming JSON, sanitises the same way as `css_props`.

- `publish()` HTML-mode INSERT (draft â†’ `pages`): adds both new columns to SELECT/INSERT.

**`public_html/js/editor.js`**

2. Existing blocks will have NULL in both new columns â€” they will inherit desktop styles

3. No changes to `pages_index`, platform DB, or any module tables.
