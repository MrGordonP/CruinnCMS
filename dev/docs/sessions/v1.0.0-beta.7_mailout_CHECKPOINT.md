## What Changed This Session

### 1. Events: External Registration Form URL
- Migration `002_external_form_url.sql`: `ADD COLUMN IF NOT EXISTS external_form_url VARCHAR(500) NULL`
- `EventController`: persists `external_form_url` on create/update
- Core template `templates/public/events/show.php`: shows external link button in place of internal registration when set (core template takes priority over module copy)
- Admin edit template: External Registration Form URL field added

### 2. Events: Related Blog Post Link
- Migration `003_related_article_id.sql`: `ADD COLUMN IF NOT EXISTS related_article_id INT UNSIGNED NULL`
- `EventController`: fetches published articles list for admin dropdown; fetches `$relatedArticle` on public show
- Core template `templates/public/events/show.php`: "Read more" link shown after share component when related article is set
- Admin edit template: Related Blog Post dropdown added

### 3. Events: Show Page 500 Fixes
- Registrations query now uses actual `event_registrations` columns: `name`, `email`, `notes`, `created_at`
- Removed all phantom column references (`member_id`, `guest_name`, `guest_email`, `registered_at`, `payment_status`)
- Breadcrumbs fixed to use positional array syntax

### 4. Mailout: Send Now
- `BroadcastController::sendNow()`: sends inline to full recipient list, skips stale pending queue rows, marks broadcast `sent`/`failed`
- `BroadcastController::resolveRecipients()`: shared helper extracted for both `queue()` and `sendNow()`
- `queue()` refactored to use `resolveRecipients()`, adds `scheduled_at` and `next_retry_at` fields
- Show page: Send Now + Queue for Sending (with optional schedule datetime) shown for `draft` and `queued` mailouts
- Edit page: same Send Now + Queue section added with save-first hint
- Route registered: `POST /admin/mailout/{id}/send-now`

### 5. Mailout: Fix — Selected Mailing List Recipients Were Ignored
- `resolveRecipients()` list branch was doing `JOIN users ON users.id = mls.user_id`, silently dropping every list contact without a linked portal account
- Fixed to read directly from `mailing_list_subscriptions` with `LEFT JOIN users` for fallback display name only
- All active contacts on the admin-selected list are now used regardless of whether they have a portal account

### 6. Mailout: Reopen Sent/Failed Mailouts as Draft
- New `BroadcastController::reopen()`: flips status back to `draft`, clears `scheduled_at`
- Show page: `Reopen as Draft` button shown for `sent` and `failed` mailouts
- Route registered: `POST /admin/mailout/{id}/reopen`
