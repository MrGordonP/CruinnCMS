## 5. Mailbox Module (new)

Full IMAP webmail module: `modules/mailbox/`

**Architecture:**
- IMAP server is permanent archive; DB stores header index only (bodies fetched live)
- Thread graph built from `In-Reply-To` / `References` headers ‚Äî no subject matching
- Per-user read receipts with three-state: unread / partial (some holders read) / all read
- Passwords AES-256-CBC encrypted at rest using instance secret
- Access derived from `organisation_officers.user_id` ‚Äî single login, role-derived mailbox access
- Admin users see all `imap_enabled` mailboxes

## CruinnCMS API ‚Äî Canonical Conventions (confirmed this session)

These are the correct patterns. Any module or template that deviates will cause errors.

### Template escaping
```php
e($value)          // CORRECT ‚Äî global helper
$this->escape($value) // WRONG ‚Äî method does not exist
```

### Flash messages
```php
// Do nothing in individual templates.
// Flash is handled globally in templates/layout.php via $flashes.
// Auth::getFlash() does not exist. Auth::getFlashes() exists but layout calls it.
```

### Breadcrumb array format
```php
['Section', '/admin/section']           // CORRECT ‚Äî positional
['label' => 'Section', 'url' => '...']  // WRONG ‚Äî named keys
```
