## What was done this session

### Template canvas editor — context rendering
- `CruinnController::edit()`: template canvas pages (`_tpl_*` slug) now load and display the published global header and footer zones above/below the canvas, matching what the preview shows. Previously the `!$isZonePage` guard suppressed all surround for template pages.
- `templates/admin/editor.php`: header/footer surround conditions updated to `(empty($isZonePage) || !empty($isTemplatePage))`.
- Zone render calls wrapped in try/catch so a render failure doesn't 500 the whole editor.

### Draft resume banner — removed
- The banner that prompted "Continue with draft / Discard & reload published" was causing a confusing loop (CSRF token sent via FormData body, not header, so the discard fetch silently failed).
- Resolved by removing the banner entirely. Draft is silently resumed (controller already loads it). Toolbar "Discard Draft" button handles explicit revert.

### Template canvas — zone block auto-seed
- When a template canvas page is opened with no blocks, the controller now seeds one `zone` block per non-header/non-footer zone from the template definition, written directly to `cruinn_blocks` (published, no draft).
- Falls back to seeding a `zone-main` block if no zones are defined.
- **Bug introduced and fixed:** seed code was initially placed before `$isTemplatePage`/`$templateZonesDef` were defined, causing 500s. Moved to correct position (after those variables are set).

### Media browser
- Folder browse, create/delete folder, upload fix, cache-busting (committed from prior work).

## Outstanding / known issues

- Zone block in template canvas renders as a narrow green strip with no width — it has no CSS width because the canvas has no layout context. The zone block shows the name badge but doesn't represent what the page layout will actually look like. A proper template canvas needs the ability to build surrounding layout (columns, sections) around the zone block.
- The "Page Content Area" placeholder shown in template preview is hardcoded in `template-preview.php` — it does not come from blocks. The editor canvas and the preview are not yet equivalent representations of the same thing.
- Zone block properties panel has no meaningful properties for controlling layout context.
