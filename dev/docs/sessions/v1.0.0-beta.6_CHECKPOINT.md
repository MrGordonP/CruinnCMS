# CruinnCMS v1.0.0-beta.6 — Checkpoint
**Date:** 2026-04-13
**Version:** v1.0.0-beta.6
**Type:** Catch-up commit — user profile, cross-domain auth, editor improvements, module renumbering
**Previous:** v1.0.0-beta.5 (`124f8f1`)

## Summary

This commit consolidates work done during the `editor-overhaul-beta4` branch recovery after a GPT session corrupted the editor. It includes several independent features and fixes that were implemented while catching up.

---

## What changed

### 1. User Profile (`/profile`)

New public-facing profile page for logged-in users:
- **Routes:** GET/POST `/profile` in `config/routes.php`
- **Controller:** `AuthController::showProfile()` and `updateProfile()`
- **Template:** `templates/public/profile.php`
- **Features:** Display name edit, email change (with uniqueness check), optional password change (requires current password verification)
- **Schema:** Updated `roles.default_redirect` from `/members/profile` to `/profile`

### 2. Cross-Domain Passthrough Tokens

Secure cross-domain admin access from the platform dashboard to instance admin panels:
- **`PlatformController::generatePassthroughToken()`** — Creates HMAC-signed token (SHA-256) with 60-second validity
- **`validatePassthroughToken()`** — Validates signature, expiry, and destination match
- **Secret:** Uses platform DB password or credential file password_hash
- **Usage:** Dashboard "Open Admin" and "ACP Settings" buttons now include `&pt={token}` parameter

### 3. Editor CSS Improvements

- **Visible block outlines:** All `[data-block]` elements have a dashed green outline (`rgba(29, 158, 117, 0.35)`) in editor mode
- **Layout container minimum sizes:** `columns`, `section`, `form`, `list`, `table` blocks have `min-height: 60px; min-width: 100px`
- **Empty container placeholders:** Show "Drop blocks here" text when empty
- **Drag-to-resize:** Active blocks have `resize: both; overflow: auto`
- **Collapsible panel sections:** CSS for `.editor-panel-section.collapsed`

### 4. CSS Class Persistence (completing beta.5 work)

- `CruinnRenderService` and `EditorRenderService` now output `class` attribute from `css_props._class`
- Skip `class` from `_attrs` to avoid duplication

### 5. Module Migration Renumbering

All modules renamed from IGAPortal-era numbers to clean `001_*_core.sql`:

| Module | Old | New |
|--------|-----|-----|
| blog | `005_content_articles.sql` | `001_articles_core.sql` |
| drivespace | `016_file_manager.sql` | `001_drivespace_core.sql` |
| events | `004_event_enhancements.sql` | `001_events_core.sql` |
| forms | `020_forms.sql` | `001_forms_core.sql` |
| gdpr | `014_gdpr.sql`, `015_deleted_accounts.sql` | `001_gdpr_core.sql`, `002_deleted_accounts.sql` |
| mailout | `032_*`, `034_*`, `035_*` | `001_mailout_core.sql` |
| membership | — | `002_membership_year.sql` |
| oauth | `013_oauth_accounts.sql` | `001_oauth_core.sql` |
| social | `007_social_media.sql` | `001_social_core.sql` |

### 6. Miscellaneous Fixes

- **ImportService:** Handles fragment files (no `<body>` wrapper) — treats entire content as body
- **UserAdminController:** Gracefully handles missing `members` table (membership module not installed)
- **CruinnController:** Fixed `apiBase` for platform context, fixed revert redirect URL
- **PlatformController:** MySQL 8 information_schema queries use uppercase column names (`TABLE_NAME` etc.)
- **Auto-import fix:** Clears doc-only blocks before re-importing to prevent accumulation
- **Platform layout:** Fixed typo (`span>Dashboard` → `<span>Dashboard`)
- **Admin layout:** Added `file_exists()` guard before `filemtime()` calls
- **User templates:** Guard `email_verified_at` column display (may not exist)
