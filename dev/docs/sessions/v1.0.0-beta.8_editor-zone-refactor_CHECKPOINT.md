## Work Done This Session

### 1. Stage 4b — Render pipeline unification (committed `d840151`)

**Goal:** Replace the parallel middleware-based zone assembly (header/footer/sidebar each
fetched separately and passed as Template globals) with a single `buildWithTemplate()` call
that walks the template block tree and injects all zones inline.

**Files modified:**

- `src/Services/CruinnRenderService.php`
  - New public method `buildWithTemplate(int $templateId, string $pageZone, array $zoneCanvasMap, ?int $pageId, ?string $injectHtml): array`
  - New public method `resolveZoneCanvasId(string $zone, ?int $templateId, ?int $pageId): ?int` — 4-level resolution (page zone_overrides → template zone_canvases → global canvas_type='zone' → legacy _slug)
  - New public method `buildZone(string $zone, ?int $templateId, ?int $pageId): ?array`
  - New private method `renderTemplateTree(...)` — walks template blocks; on zone blocks injects page content or zone canvas blocks
  - `buildCssForTemplate()` — wrapped in try/catch for pre-012 fallback
  - `hasPublishedTemplate()` — queries via `template_id` column

- `src/Controllers/PageController.php`
  - `home()` and `show()` both collapse to a single `buildWithTemplate()` call
  - New private `buildZoneCanvasMap()` — builds `[zoneName => canvasPageId]` for all template zones except the page's own zone
  - Removed: `setZoneGlobals()`, `resolveZoneRender()`, `resolveSidebarRender()`

- `src/App.php`
  - Zone globals middleware block removed entirely

- `templates/layout.php`
  - Reduced to HTML document shell: `<head>` + skip link + flash messages + `<?= $_pageContent ?>` + scripts
  - All structural wrappers (`<header>`, `<footer>`, `<aside>`, `site-body-wrap`) removed

### 5. Unconsumed zones — hack then proper fix

**Hack (committed `9746244`, immediately superseded):** Added `_consumedZones` tracker to
`CruinnRenderService` to render zone canvases outside the template tree when no zone block
existed. Correctly identified as a cludge.

**Proper fix — migration 013 (committed `9a6fba4`):**

- `migrations/core/013_backfill_template_zone_blocks.sql` — stored procedure that:
  1. For each global zone canvas (`canvas_type='zone'`), for each template not already wiring that zone:
     - Updates `page_templates.zone_canvases` JSON with `{zone_name: canvas_id}`
     - Updates `page_templates.zones` JSON to include the zone name
     - Inserts a `zone` block into the template's block tree:
       - Header-type zones (`header`, `topbar`, `navbar`, `banner`): sort_order=0,
         shifts all existing root blocks up by 1 first
       - All other zones (footer, sidebar etc.): sort_order=32767
  - `_consumedZones` property and all associated code reverted from `CruinnRenderService`

**Key design note:** Zone block `block_config` key is `zone_name` (not `zone`) — matches
`renderTemplateTree()`'s `$tCfg['zone_name'] ?? 'main'` lookup.

**Block ID format:** `CONCAT('zm', LPAD(template_id, 5, '0'), LPAD(canvas_id, 5, '0'))` —
12 chars, unique per template×canvas combination, fits VARCHAR(20).

### 6. Editor context zone fixes (committed `e9bb4f0`)

After migration 012, two editor variables were broken because they queried template blocks
via `page_id = canvas_page_id` (old path):

- **`$templateZones`** (the zone picker in the editor): was derived by querying zone blocks
  from `pages WHERE page_id = canvas_page_id`. Now reads from `$tplRow['zones']` JSON
  directly — the authoritative source.

- **`$templateCanvasHtml`** (template layout structure preview shown below the canvas):
  was built with `hasPublished(canvas_page_id)` + `buildHtml(canvas_page_id)`. Now uses
  `hasPublishedTemplate($templateId)` + `buildWithTemplate($templateId, 'main', [])`.
  Renders the template shell with empty zone slots as a structural preview.

**File modified:** `src/Controllers/CruinnController.php` — `edit()` method,
`templateCanvasHtml`/`templateZones` block in the non-template-page branch.

### Stage 4c remaining gaps

1. **Item 17 — Back/breadcrumb navigation:** When an editor navigates from a page canvas
   to a context zone canvas (header/footer), there is no back button or breadcrumb to return
   to the originating page. The forward navigation (click context zone → edit that zone) works
   via `href` on the `<a>` element. The return path is not implemented.

2. **`openEditor()` legacy code:** The no-page-loaded editor state in `CruinnController::openEditor()`
   still queries `_header`/`_footer` slugs and `canvas_page_id` to build `headerPages`/`footerPages`.
   This should be replaced with zone canvas model queries, consistent with `edit()`.

3. **Sidebar context unification:** Sidebar is handled as a separate legacy path
   (`$sidebarContextHtml`, `$sidebarContextCss`, `$sidebarContextPageId`) rather than as
   a member of `$contextCanvases`. Should be unified — sidebar becomes another `contextCanvases`
   entry with `position='right'` or similar.
