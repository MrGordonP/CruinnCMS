# v1.0.0-beta.14 Checkpoint — Zone Canvas Reference Migration

**Date:** 20 May 2026  
**HEAD:** `30d368f`  
**Previous:** `57dd9f8` (v1.0.0-beta.13)

---

## Summary

Eliminated dual-control antipattern in template zone architecture by migrating canvas references from `page_templates.zone_canvases` JSON column into zone block `block_config.canvas_page_id` field. Zone blocks now own their canvas references directly.

**Status:** Data model migration complete, PHP refactored, backward compatibility preserved. Editor UI for zone canvas assignment pending (deferred to next version).

---

## Changes

### Migration 020: Zone Block Canvas References

**File:** `migrations/core/020_zone_block_canvas_references.sql`

Backfills `canvas_page_id` from `page_templates.zone_canvases` JSON into `pages.block_config` for all zone blocks. Migrates existing template canvas assignments without data loss.

```sql
UPDATE pages p
JOIN page_templates pt ON p.template_id = pt.id
SET p.block_config = JSON_SET(
    COALESCE(p.block_config, '{}'),
    '$.canvas_page_id',
    CAST(JSON_EXTRACT(pt.zone_canvases, CONCAT('$."', p.block_config->>'$.zone_name', '"')) AS UNSIGNED)
)
WHERE p.block_type = 'zone'
  AND pt.zone_canvases IS NOT NULL
  AND JSON_EXTRACT(pt.zone_canvases, CONCAT('$."', p.block_config->>'$.zone_name', '"')) IS NOT NULL;
```

### CruinnRenderService: Self-Resolving Zone Rendering

**File:** `src/Services/CruinnRenderService.php`

- **`buildWithTemplate()`**: Now self-resolves zone canvas map from template zone blocks
  - Extracts zone blocks from template
  - Reads `canvas_page_id` from each zone block's `block_config`
  - Applies page-level `zone_overrides` on top
  - Fetches zone canvas blocks and renders merged tree
- **Removed:** `resolveZoneCanvasId()`, `buildZone()` (dead code from old architecture)

### PageController: Removed Zone Canvas Map Building

**File:** `src/Controllers/PageController.php`

- **Removed:** `buildZoneCanvasMap()` method (no longer needed)
- **Updated:** `home()`, `show()` — removed zone canvas map parameter from `buildWithTemplate()` calls
- Template zone resolution now handled entirely within `CruinnRenderService`

### CruinnController: Context Canvas Building from Zone Blocks

**File:** `src/Controllers/CruinnController.php`

- **Context canvas building:** Reads zone blocks from template, resolves `canvas_page_id` with fallback to legacy `zone_canvases` JSON
- **Backward compatibility:** Query checks `zone_canvases` column if `canvas_page_id` not populated
- **Resolution order:** page override → zone block `canvas_page_id` → legacy `zone_canvases` JSON → global zone canvas

```php
$zoneQuery = "SELECT block_config 
              FROM pages 
              WHERE template_id = ? 
              AND block_type = 'zone' 
              AND block_config->>'$.zone_name' = ?";
$zoneBlock = $db->fetchOne($zoneQuery, [$pageData['template_id'], $zoneName]);

if ($zoneBlock && isset($zoneBlock['block_config']['canvas_page_id'])) {
    return (int)$zoneBlock['block_config']['canvas_page_id'];
}

// Fallback to legacy zone_canvases JSON
$legacyCanvases = json_decode($templateData['zone_canvases'] ?? '{}', true);
if (isset($legacyCanvases[$zoneName])) {
    return (int)$legacyCanvases[$zoneName];
}
```

## Database Schema Impact

**No schema changes** — uses existing `pages.block_config` JSON column. Migration 020 populates data only.

**New block_config structure for zone blocks:**

```json
{
  "zone_name": "header",
  "canvas_page_id": 1,
  "collapsed": false
}
```

**Legacy `page_templates.zone_canvases` preserved** for backward compatibility during transition period.

## Backward Compatibility

- **Fallback logic:** If `canvas_page_id` not found in zone block, falls back to `zone_canvases` JSON column
- **No breaking changes:** Existing templates continue to work pre-migration
- **Safe migration path:** Can run migration 020 independently without code deploy
