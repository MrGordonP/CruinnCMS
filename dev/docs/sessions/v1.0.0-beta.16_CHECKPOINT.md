# v1.0.0-beta.16 Session Checkpoint

## What Changed

### Bug Fix: Dashboard listing empty
- **Root cause 1:** `'widget-dashboard'` was not a valid value in the `pages_index.canvas_type` ENUM, causing all dashboard inserts to throw an exception.
- **Root cause 2:** The catch-path fell back to a slug-based insert (`dashboard-*`) and a query with `LIKE '\_dashboard\_%'` that never matched.
- **Fix:** Migration `027_canvas_type_widget_dashboard.sql` adds `'widget-dashboard'` to the ENUM and backfills existing dashboard rows.
- `DashboardService::listDashboardCanvases()` — removed slug fallback, now single clean query on `canvas_type = 'widget-dashboard'`.
- `SiteBuilderController::builderCreateDashboard()` — removed try/catch slug fallback.

### New Feature: Three-panel Subjects Workspace
- Route `GET /admin/subjects/{id}` added to `config/routes.php`.
- `SubjectController::adminList()` now renders the three-panel workspace with `subject=null`.
- `SubjectController::adminView(string $id)` added: fetches ancestors (walking parent_id chain), children, all four content types via bridge, parentSubjects for settings form.
- Template `templates/admin/subjects/workspace.php` created: recursive tree, breadcrumb chain, content summary, compact settings form.

### Schema: subject_content polymorphic bridge table
- Migration `028_subject_content_bridge.sql` creates `subject_content(subject_id, item_type, item_id)` bridge table.
- Migrates any existing non-NULL `subject_id` values from `articles`, `events`, `files`, `folders`.
- Drops `subject_id` FK constraints, indexes, and columns from all four tables.
- All steps guarded with `information_schema` checks + PREPARE/EXECUTE conditional DDL.

### ArticleController (`modules/blog/src/Controllers/ArticleController.php`)
- `adminList()`: filter changed from `a.subject_id = ?` to `EXISTS (SELECT 1 FROM subject_content ...)`. List query now uses bridge LEFT JOIN + GROUP_CONCAT for `subject_titles`.
- `adminNew()`: passes `articleSubjectIds = []` to template.
- `adminCreate()`: removed `subject_id` from insert. Syncs `subject_content` after insert. Passes `$subjectIds[0] ?? 0` to forum provisioner.
- `adminEdit()`: fetches `articleSubjectIds` from `subject_content`, passes to template.
- `adminUpdate()`: removed `subject_id` from update. DELETE + re-INSERT sync of `subject_content`. Passes first subject ID to forum provisioner.
- Public `buildBlogListData()` and `findPublishedArticleBySlug()`: replaced `LEFT JOIN subjects s ON a.subject_id = s.id` with bridge JOIN.

### Article templates
- `edit.php`: single `<select name="subject_id">` → `<select name="subject_ids[]" multiple>`, checked using `in_array` against `$articleSubjectIds`.
- `index.php`: `$article['subject_title']` → `$article['subject_titles']`.

### EventController (`modules/events/src/Controllers/EventController.php`)
- `adminCreate()`: removed `subject_id` from insert. Syncs `subject_content` after insert.
- `adminEdit()`: fetches `eventSubjectIds` from `subject_content`, passes to template.
- `adminUpdate()`: removed `subject_id` from update. DELETE + re-INSERT sync of `subject_content`.
- `normaliseSubjectId()` helper removed.

### Event template
- `edit.php`: single `<select name="subject_id">` → `<select name="subject_ids[]" multiple>`, checked using `in_array` against `$eventSubjectIds`.

### FileManagerController (`modules/drivespace/src/Controllers/FileManagerController.php`)
- `index()`: file listing query now uses correlated subquery for `subject_title`. Fetches `$currentFolderSubjectIds` from bridge. Passes to template.
- `upload()`: removed `subject_id` from file insert. Syncs `subject_content` for `item_type='file'` after insert.
- `show()`: file query uses correlated subquery for `subject_title`.
- `createFolder()`: removed `subject_id` from folder insert. Syncs `subject_content` for `item_type='folder'`.
- `updateFolder()`: removed `subject_id` from folder update. If `subject_ids` field present in POST, DELETE + re-INSERT sync of `subject_content`.
- `folderInfo()` and `fileInfo()` AJAX endpoints: removed `LEFT JOIN subjects` in favour of correlated subquery.

### Drivespace templates
- `upload.php`: subject single-select → multi-select (`name="subject_ids[]"`).
- `index.php`: create-folder modal and edit-folder modal both converted to multi-select. Edit modal pre-selects from `$currentFolderSubjectIds`.
