## What Changed

### 1. Per-Instance Block Type Activation

Mirrors the existing module activation model. Block types now have a lifecycle state per instance (`discovered` / `active` / `offline`), stored in a new `block_type_config` table.

**Schema:**
- `schema/instance_core.sql` — added `block_type_config` table (`slug`, `status`, `updated_at`)
- `migrations/core/030_block_type_config.sql` — migration for existing instances (apply via ACP `/admin/maintenance/migrations`)

**BlockRegistry (`src/BlockTypes/BlockRegistry.php`):**
- New properties: `$statuses`, `$dbChecked`
- `load()` rewritten — activation-aware. Only `require`s `definition.php` for `active` slugs. Falls back to loading all if `block_type_config` table doesn't exist (safe on fresh install)
- New public methods: `statusOf()`, `isActive()`, `allDiscovered()` (returns all on-disk slugs for ACP browse)

**ACP Block Types page:**
- `src/Admin/Controllers/AcpSystemController.php` — three new methods: `blockTypes()`, `activateBlockType()`, `deactivateBlockType()`
- `config/routes.php` — three new routes: `GET /admin/settings/block-types`, `POST /admin/settings/block-types/{slug}/activate`, `POST /admin/settings/block-types/{slug}/deactivate`
- `templates/admin/settings/block-types.php` — new ACP template using shared `.module-card` styles
- `templates/admin/settings/_tabs.php` — Block Types tab added after Modules tab

---

### 2. Dynamic Editor Palette

The editor palette is now fully driven by block type definitions — no hardcoded group maps.

- `templates/admin/editor.php` — palette loop replaced with dynamic PHP loop collecting `$_paletteGroups` and `$_paletteGroupLabels` from `BlockRegistry::all()` in first-seen order
- All 12 palette block type definitions updated to include `group`, `group_label`, and `palette_entries` keys:
  - `layout` / Layout & Containers: element, section, columns, table
  - `text` / Text: heading, text, list
  - `media` / Media: image, gallery
  - `dynamic` / Data & Dynamic: html, dynamic-include, data-list
- Zone button remains a hardwired special case within the layout group

### 5. Core Module Registration via `ModuleRegistry::registerCore()`

Core platform modules (`subjects`, `accounts`) are now declared using the same contract as drop-in modules but without being tracked in `module_config`.

**ModuleRegistry (`src/Modules/ModuleRegistry.php`):**
- New `registerCore(array $def)` — calls `register()` then immediately sets `$statuses[$slug] = 'active'`, bypassing the `module_config` table check

**App.php (`src/App.php`):**
- After `ModuleRegistry::load()`, calls `ModuleRegistry::registerCore()` for:
  - `subjects` — ACP section at `/admin/subjects`, two `widgets` closure entries: `subjects_quick_link` (quick link tile) and `subjects_status_summary` (counts card with live DB queries)
  - `accounts` — ACP section at `/admin/users`

---

### 6. `core_fragment` Source Type Eliminated

`core_fragment` was a half-finished concept for surfacing platform content in the editor. Its two renderers now live as proper module widgets on the `subjects` core module registration above.

**Files cleaned:**
- `src/BlockTypes/dynamic-include/definition.php` — `core_fragment` branch removed, `core_fragment_key` removed from `unset()` calls, docblock updated
- `templates/admin/editor.php` — `data-core-fragment-styles` attribute removed; `<option value="core_fragment">` removed; `#prop-dyn-core-fragment-row` and `#prop-dyn-core-fragment-key-row` sub-panels removed
- `public_html/js/editor.js` — `CORE_FRAGMENT_CATALOG` constant removed; `findCoreFragmentModule()` and `populateCoreFragmentOptions()` removed; `core_fragment` handling removed from `resolveDynamicIncludeSourceType`, `syncDynamicIncludeContentUi`, `loadProps` event wiring, `buildPhpIncludeVarRows`, and `refreshPhpIncludePreview`; `core_fragment_key` removed from config key filter lists
- `src/Controllers/CruinnController.php` — `loadCoreFragmentStyles()`, `fragmentStyles()`, and `saveFragmentStyles()` removed; three `coreFragmentStyles` render data assignments removed
- `config/routes.php` — four `fragment-styles` routes removed (`GET`+`POST` for both `/admin` and `/cms` prefixes)
