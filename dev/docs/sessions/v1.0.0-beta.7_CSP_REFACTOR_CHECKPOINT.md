## Hard Rules (Contract ‚Äî Never Relaxed)

| Rule | Detail |
|------|--------|
| No inline `<script>` blocks | External `src=` only |
| No `on*` HTML attributes | Event listeners bound in external JS |
| No `javascript:` URLs | Replace with button + listener |
| No JS frameworks | Vanilla only |
| No same-origin exceptions | Every script must be `src="/js/..."` |
| No monolithic growth | One file per page/concern |
| No bundling for convenience | Load only what each page needs |
| No feature changes during extraction | Behavior parity only |
| PHP ‚Üí JS data handoff | `data-*` attributes on root containers or hidden inputs |

**Definition of done per file:**
- Zero inline `<script>` blocks in template PHP
- Zero `on*` attribute handlers in template PHP
- Zero `javascript:` links in template PHP
- CSP-safe under `script-src 'self'`
- Behavior parity confirmed

**Slice execution rhythm:**
1. Read the PHP template in full.
2. Identify every inline `<script>` block and `on*` attribute.
3. State intent (what moves, what IDs/data-attributes are added). Wait for approval.
4. Extract to a new page-scoped JS file under `public/js/platform/` or `public/js/admin/`.
5. Remove inline code from PHP. Add stable `id=` and `data-*` hooks where needed.
6. Verify with VS Code error checker ‚Äî zero PHP errors.
7. Commit: `refactor(scope): externalize inline JS in <file>` and push.
8. Move to next file only after push is confirmed clean.

### `src/Platform/Controllers/PlatformController.php`
**Bug fix (public path resolution):**  
Source browser was resolving all paths against `CRUINN_ROOT/public/` ‚Äî on cPanel deploys `public/` is `CRUINN_PUBLIC` (the web root), which is a different directory.

**Changes:**
- Added `resolveSourcePath(string $relPath): ?array` ‚Äî routes `public/*` through `CRUINN_PUBLIC`, all others through CruinnCMS root
- Added `resolveSourceDirPath(string $relDir): ?array` ‚Äî same for directories
- `buildSourceFileGroups()` ‚Äî CSS/JS groups now scan `CRUINN_PUBLIC`; added **JS ‚Äî Platform** group for `public/js/platform/`
- `buildSourceFileTree()` ‚Äî `public` tree root now points to `CRUINN_PUBLIC`
- All five source methods (`platformSource`, `platformSourceSave`, `platformSourcePull`, `platformSourcePreview`, `platformSourcePullDir`) now use the centralised resolvers
