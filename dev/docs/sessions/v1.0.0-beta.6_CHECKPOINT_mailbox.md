## Phase 1: Production Bugfixes (geology.ie)

### Schema Context

The mailbox module previously stored credentials in `organisation_officers` directly. This session's work was predicated on a new schema introduced at the tail of the previous session:

- **`mailboxes`** — dedicated credential store (`id`, `label`, `email`, `imap_*`, `smtp_*`, `imap_last_uid`, `enabled`)
- **`mailbox_access`** — per-grant access control (`mailbox_id`, `user_id`, `officer_position_id`)

Migration: `CruinnCMS/modules/mailbox/migrations/003_mailboxes_table.sql`

---

### Bug 1 — Router: dots/plus in IMAP folder names (`bb9fb3a`)

**Problem:** IMAP folder names like `INBOX.Sent` or `[Gmail]/Sent Mail` contain `.`, `+`, `%` which the router's route segment regex didn't allow. Navigating to such folders gave a 404.

**Fix:** `src/Router.php` — route segment pattern extended to allow `.`, `+`, `%`, and encoded characters.

---

### Bug 2 — `mail()` fallback for send (`d842909`, `88cbb77`)

**Problem:** geology.ie doesn't have PHPMailer. `MailboxService::send()` assumed PHPMailer was always available and crashed.

**Fix:** `MailboxService::send()` now checks if PHPMailer class exists; falls back to PHP `mail()`. Added `parseAddressList()` helper to parse `Name <email>` format. PHP `mail()` receives the bare email address as envelope; the `Name <email>` string goes into the `To:` header only.

---

### Bug 3 — `pruneReads` / `pruneTagMap` TypeError (`0679e68`)

**Problem:** PDO returns all values as strings. `pruneReads()` and `pruneTagMap()` compared mailbox IDs with strict equality against an int and silently pruned nothing (or crashed depending on PHP version).

**Fix:** `(int) $mailbox['id']` cast at call sites in `MailboxController`.

---

### Bug 4 — Missing UID crash in `fetchBody` (`2c1f4ff`)

**Problem:** If a message UID no longer existed on the IMAP server (deleted, moved), `fetchBody()` threw a `RuntimeException` and resulted in an unhandled 500.

**Fix:** `MailboxController::message()` wraps `fetchBody()` in try/catch, redirects to the folder with a flash error message on failure.

---

### Bug 5 — `move()` returning JSON for a full-page POST (`801076f`)

**Problem:** `MailboxController::move()` was returning `json(['ok' => true])` for a normal form POST, resulting in raw JSON appearing in the browser.

**Fix:** `move()` now redirects to the source folder after the move. JSON response only used when `X-Requested-With: XMLHttpRequest` header is present.

---

### Bug 6 — `syncFolder` IMAP namespace error + wrong table (`1d33162`)

**Problem:** `uidSearch()` in `MailboxService::syncFolder()` threw an uncaught exception when the IMAP server returned an unexpected namespace response (Plesk/Dovecot issue). Also: the `imap_last_uid` column update was targeting `organisation_officers` (old schema) instead of `mailboxes`.

**Fix:** `uidSearch()` call wrapped in try/catch inside `syncFolder()`. Table reference corrected to `mailboxes`.

---

### Bug 7 — `MailboxAdminController` parse error + UTF-8 BOM (`addc13b`, `6aad18d`)

**Problem:** Orphaned duplicate method definitions were left outside the class closing brace after a previous refactor, causing a PHP parse error. Additionally the file had a UTF-8 BOM which broke `declare(strict_types=1)` on the server's PHP.

**Fix:** Truncated file to remove orphaned methods; BOM stripped using `[System.IO.File]::WriteAllBytes`.

---

### Bug 8 — Admin index template: `imap_enabled` vs `enabled` (`dc8871f`)

**Problem:** New `mailboxes` table uses column `enabled`; the admin index template was checking `$mb['imap_enabled']` (old column name). Toggle checkbox was always unchecked.

**Fix:** Template updated to use `$mb['enabled']`.

---

### Bug 9 — `position` undefined in admin index (`90de1d3`)

**Problem:** Admin controller's `index()` query was missing the `label AS position` alias after the schema migration refactor. Template uses `$mb['position']`.

**Fix:** Alias restored to the SELECT.

---

### Bug 10 — Orphaned old panel markup in admin index template (`d6bb7cc`)

**Problem:** After the three-panel admin redesign, old panel markup was appended after the new content in `templates/admin/mailbox/index.php`, causing duplicate UI.

**Fix:** Old markup removed.

## Phase 2: Officers Admin — Inline Mailbox Assignment (`d52b0c1`)

### What was built

The `/admin/organisation/officers` page now has a **Mailbox Access** section in each officer's edit row. Officers can be granted access to one or more mailboxes directly from the officers admin page — no need to visit the mailbox admin separately.

### Files changed

**`modules/organisation/src/Controllers/OrganisationAdminController.php`**
- `officers()`: now fetches `$allMailboxes` and `$mailboxGrantsByOfficer` when the mailbox module is active.
- New `assignMailbox(int $id)`: POST — inserts into `mailbox_access` for the given officer position.
- New `revokeMailbox(int $id, int $grant_id)`: POST — deletes from `mailbox_access` by grant ID, scoped to the officer position.

**`modules/organisation/module.php`**
- Routes added: `POST /admin/organisation/officers/{id}/mailbox/assign` and `POST /admin/organisation/officers/{id}/mailbox/{grant_id}/revoke`.

**`modules/organisation/templates/admin/organisation/officers.php`**
- Dead ✉️ mailto link removed from officer rows.
- **Mailbox Access** section added inside each officer's edit row: current grants as chips with × revoke buttons; Assign dropdown for unassigned mailboxes.

---

## Phase 3: Mail Front-End UX Redesign (`f76e943`)

### What was changed

**`/mail` landing page** — was a simple table listing accessible mailboxes. Now redirects directly to the first accessible mailbox's INBOX. The standalone list page is gone.

**`messages.php` sidebar** — replaced the single-mailbox folder nav with a **unified multi-mailbox sidebar**:
- All accessible mailboxes listed in order (`sort_order ASC, label ASC`).
- Each mailbox is a clickable header (links to its INBOX).
- The currently active mailbox expands inline to show all its folders (indented) and the Compose button.
- Clicking a different mailbox navigates to its INBOX; the shell remains consistent.
- "← All Mailboxes" footer link removed — all mailboxes are always visible.

**`sort_order` column on `mailboxes` table**
- Migration `004_mailbox_sort_order.sql`: `ALTER TABLE mailboxes ADD COLUMN sort_order INT UNSIGNED NOT NULL DEFAULT 0 AFTER enabled`.
- `MailboxService::getAccessibleMailboxes()`: orders by `sort_order, label`; `label` and `sort_order` added to all SELECT lists.
- `MailboxAdminController`: `newForm()`, `credentialsPanel()`, and `collectCredentialFields()` all handle `sort_order`.
- `credentials-panel.php`: "Sort order" number input added to the Identity section.
