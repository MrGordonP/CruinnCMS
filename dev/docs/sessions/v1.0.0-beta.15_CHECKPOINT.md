Date: 2026-05-21

## Checkpoint scope

This checkpoint captures the beta.15 template/layout work plus a fast follow-up fix to expose Blog list system-page binding in the existing Modules settings UI and sync `system_pages` automatically.

## Commits included in this checkpoint

- 6e67c0f fix(templates): align layout-driven template UX and editor panel behavior [v1.0.0-beta.15]
- b77bbb0 fix(site-builder): add template-layout create flow and 3-panel templates UI [v1.0.0-beta.15]
- Current checkpoint also includes: fix(system-pages): blog list page mapping via module settings UI [v1.0.0-beta.15]

## What changed

### 1) Template Layout management is now first-class

- Added dedicated create/delete endpoints for Template Layout pages.
- Added server-side create logic for standalone layout pages (`canvas_type=template-shell`).
- Added server-side delete guardrails to prevent deleting layouts in use.

### 2) Templates screen moved to a 3-panel layout

- Left panel now separates `Page Templates` and `Template Layouts`.
- Main panel lists records for the selected section.
- Right panel now shows section-relevant information instead of mixed controls.

### 3) Page Templates now follow layout-driven zone ownership

- Removed zones JSON input from page template creation UI.
- Page template creation now requires selecting a Template Layout.
- Page template zones are derived from zone blocks on the selected Template Layout.
- Initial template zone assignment blocks are synced from that layout.

### 4) Right-panel UX cleanup

- Page Templates:
  - Right panel defaults to selected-template details.
  - Create form is moved behind the top `+ New Template` action.
  - Added cancel to return from create mode to details mode.
- Template Layouts:
  - Right panel now shows selected-layout details and actions.
  - Creation remains top-toolbar driven.

### 5) Editor and navigation polish

- Editor sidebar links now route directly to Template Layout management/create view.
- Page Settings accordion hidden when editing Template Layout pages.

### 6) Compatibility fix for older instance schemas

- Removed `pages_index.updated_by` writes from new layout/dashboard page inserts in `SiteBuilderController` to avoid SQLSTATE 42S22 on instances where the column does not exist.

### 7) Action consistency

- Page templates now always show an Edit action in list view.
- Edit action routes through ensure-canvas flow so templates without a canvas can still be edited immediately.

### 8) Blog list system-page mapping now has UI path

- Added `Blog List Page ID` field to Blog module `settings_schema`, visible in ACP Modules > Blog > Settings.
- Saving Blog module settings now syncs `system_pages` for `blog.list`:
  - Valid page ID: upsert mapping to that `pages_index.id`
  - Blank/invalid: remove `blog.list` mapping
- Added safety checks:
  - Reject mapping if selected page ID does not exist
  - Reject mapping if selected page is already assigned to a different system key
- Existing runtime behavior in `ArticleController::index()` remains unchanged; it now becomes UI-configurable through settings save.
