# CruinnCMS v1.0.0-beta.4 — Checkpoint
**Date:** 2026-04-10
**Version:** v1.0.0-beta.4
**Type:** Editor overhaul — kill Editor 2, platform editor improvements, code view for CSS files
**Previous:** v1.0.0-beta.3 (`bc70dd2`)

---

## What changed

### 1. Editor 2 completely removed

The old "block editor" (Editor 2) was a parallel system that never worked properly. All traces removed:

- **Deleted files:** `public/js/admin/block-editor/core.js`, `drag.js`, `properties.js`, `undo.js`, `public/js/admin/site-editor.js`, `public/css/admin-block-editor.css`, `public/css/admin-site-editor.css`, `templates/admin/components/block-editor.php`, `templates/admin/components/block-props-panel.php`
- **Cleaned references:** `public/js/admin/index.js` — removed Editor 2 init chains; `templates/admin/layout.php` — removed Editor 2 CSS includes; `templates/admin/pages/edit.php` — removed Editor 2 container markup
- **Stubbed `content_blocks` queries:** `AdminPageController`, `AdminImportController`, `SiteBuilderController`, `AcpSystemController`, `MediaController` — any runtime reference to the dropped `content_blocks` table now returns empty arrays or skips gracefully

### 2. Council templates removed

Deleted all council workspace templates (`templates/council/`) — these were IGAPortal-era remnants with no routes or controllers in CruinnCMS.

### 3. Platform PAGES nav — filter internal pages

`PlatformController` now filters `_cms_src_` prefixed pages from the PAGES nav in the editor left panel. These are internal page records created by the `?file=` handler and should not appear as navigable pages.

### 4. CSS files available in platform editor nav

New "CSS FILES" nav group in the editor left panel populated via `glob(CRUINN_PUBLIC . '/css/*.css')`. Each link opens the CSS file in the editor using the `?file=` handler.

### 5. `?file=` handler — cPanel path resolution fix

The `?file=public/css/editor.css` handler was resolving paths against `$rcRoot` (the engine root), but on cPanel the `public/` directory contents live at `CRUINN_PUBLIC` (web root), not at `$rcRoot/public/`. Fixed both the `?file=` handler and `resolveRenderFilePath()` to route `public/*` subpaths through `CRUINN_PUBLIC`.

**Lesson:** On cPanel split installs, `public/*` paths must always resolve via `CRUINN_PUBLIC`, never via engine root + `/public/`.

### 6. Code view for non-PHP/HTML files

Files with extensions other than `.php`, `.html`, `.htm` (e.g., `.css`, `.js`) now:
- Set `startInCodeView = true` so the editor opens directly in code view
- Skip block import in `ImportService` (no point decomposing CSS into blocks)
- Load raw file content into `data-html-content` for the textarea

### 7. Code view toggle fix

`exitCodeView()` had a guard `if (_htmlPageMode) { return; }` that prevented toggling out of code view for file-mode pages. Removed — the toggle now works in both directions.

### 8. Code view CSS layout (in progress)

Added `:has(#editor-code-area)` CSS rules to remove `max-width: 1200px`, `box-shadow`, and `padding: 2rem` constraints from `#editor-canvas` and `#editor-canvas-wrap` when the code textarea is present. Added `width: 100%` and `height: 100%` to `#editor-canvas` in code view mode. This is partially working — the textarea fills the column but further CSS refinement needed.

### 9. Full file path in toolbar

When editing a file via `?file=`, the toolbar now shows the full relative file path (e.g., `public/css/editor.css`) instead of just the page slug.

### 10. Block tree and properties panel scroll constraints

Applied CSS to constrain the block tree (`#editor-tree-section`) and properties panel (`#editor-props`) to their grid slots with independent scrolling:
- `#editor-left`: flex column, `overflow: hidden`, `height: 100%`
- `#editor-tree-section`: `flex: 1`, `min-height: 0`, `overflow-y: auto`
- `#editor-props`: `overflow-y: auto`, `height: 100%`
- `#editor-body`: `grid-template-rows: 1fr`, `overflow: hidden`

**Status:** CSS applied but may need further debugging — the height chain from platform layout through the grid hasn't been fully verified in browser.

### 11. Miscellaneous

- `#editor-wrap` changed from `min-height: 100%` to `height: 100%`
- `#editor-canvas-wrap` added `min-width: 0` to prevent grid blowout
- `public/css/admin-base.css` — removed `overflow: hidden` from editor container
- Platform CSS overrides in `editor.css` for `.platform-wrap`, `.platform-sidebar`, etc. to neutralise platform chrome when imported as blocks

## Known issues / remaining work

1. **Code view width** — CSS `:has()` rules are in place but the textarea may still not fill the full column in all cases. Needs browser verification.
2. **Block tree scroll** — CSS applied but user reports it still doesn't constrain. May need the full height chain from platform layout debugged.
3. **Properties panel scroll** — Same situation as block tree.
4. **CSS content in `data-html-content`** — Raw CSS is dumped into the attribute without HTML-encoding, which will break on any CSS containing `"` or `<`. Needs `htmlspecialchars()` escaping.
5. **Code view save for CSS files** — The save mechanism posts to `/admin/template-editor/edit` which may not handle non-template files correctly.
6. **Table renames** — `cruinn_blocks` → `blocks` not yet done.
7. **Overlay rendering order** — Z-index issues not addressed.
